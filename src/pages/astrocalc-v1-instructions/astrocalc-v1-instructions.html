AstroCalc v1 Instructions with Tutorial

Initial State

AstroCalc uses terminology familiar to computer programmers. Commands tell the computer what action to take (those are the blue buttons.) The Accumulator holds the result of the last command. Memory remembers the value of the Accumulator whenever you click the Store button.

The Accumulator and Memory are empty in the initial state.

Only three commands are available from here: Observatory, Universal Time, and Coordinates. The remaining commands are unavailable because they operate on memory. After you begin storing results those commands will become available.

Observatory, Universal Time, and Coordinates are always available because they simply require raw numbers from you. The other commands operate on the results of these. It is a good idea at this time to understand dependencies.

Dependencies

  • Local Time depends on Observatory and Universal Time.
  • Precession depends on Coordinates and Universal Time.
  • Transformation depends on Coordinates and Local Time.

Tutorial

This morning I watched sunrise and wondered what it looked like from Kitt Peak National Observatory in Arizona. By the time I got to it the Sun was about 25 degrees above my horizon here near New York City. My intuition told me that the Sun would be close to rising in Arizona but how close I did not know until running this calculation.

This is a screenshot of a program called MICA, the Multiyear Interactive Computer Almanac, a product from the United States Naval Observatory.

From their Introduction: "MICA is a software system created especially for astronomers, surveyors, meteorologists, navigators, and others who regularly need accurate information on the positions and motions of celestial objects. It provides high-precision astronomical data for a variety of astronomical objects."

You can purchase it for $29.95 from Willmann-Bell, Inc.

The primary difference between an Almanac and AstroCalc is the way the data is organized. If you ever purchased the Old Farmers' Almanac at the checkout of your local grocery store then you are familiar with how data is organized. It is in tables. If you are interested in the time of sunrise in your town on a particular date then you need to interpolate since the almanac is not tailored for you personally. Internally, MICA is organized similarly. AstroCalc, on the other hand, uses formulas to do its work. I am sure that MICA uses formulas also but it performs the heavy computations beforehand whereas AstroCalc performs it on demand. AstroCalc is not as precise as MICA but it does not need to for most applications.

For this tutorial, the date is October 14, 2019 and the time is 9:59:07 local time on the U.S. east coast, daylight savings time. That is equivalent to 13:59:07 Universal Time.

Step 1

Use an Internet search or map service to find the latitude and longitude of Kitt Peak.

Click the Observatory command button.

Fill in the form fields and click "OK".

You can see that the result of the Observatory command is placed in the Accumulator:

This is a good time to emphasize that the Accumulator is volatile. It changes after each command so it is a good idea to save it to Memory by clicking the Store button. Not only will it save your work but it will also make it available to other commands.

Save the Observatory results to memory.

Click the Store command button.

Accept the default memory location and then click "OK".

Tip: Use the default memory location for all store operations. You will save a lot of time. After developing experience with AstroCalc you may develop advanced solutions that may benefit from a different name. For now however choose the default.

Here you can see the result of the Store command:

For now on I will rely on you to store the result of commands. I'll simply show the result.

Step 2

Set Universal Time. The Earth's Prime Meridian is the great arc from North to South pole that passes through Greenwich, England. Think of Universal Time as the local time at the Prime Meridian. All calculations begin with it, even the position of the planets in their orbits.

Click the Universal Time command button.

Fill in the fields and click "OK". (If you want to follow this tutorial and achieve the same results then you need to use these same values.) After clicking "OK" then click the Store button to save the result to Memory. Use the default memory location.

Tip: AstroCalc pre-populates this form by reading your computer's date and time. This is intended to save you time by anticipating that you are interested in where things are right now.

Notice that the initial date and time will be different than your computer time if you don't live along the Prime Meridian. For example I live in the New York City area. Depending on the season my local time is either four or five hours behind Universal Time. Keep this in mind if you want to specify some other date and time in your location. You will need to adjust your local time by your time zone's Zone Correction in order to derive Universal Time.

Results Comparison

Let's compare the results of our calculations to MICA:

Measurement MICA AstroCalc Comparison
Universal Time 2019 Oct 14 13:59:07.0 2019-10-14 13:59:07 Same
Julian Date 2458771.082720 2458771.08272 Same
Greenwich Mean Sidereal Time (GMST) 15:30:26.2084 15:30:26 Same See note 1
Local Mean Sidereral Time (LMST) 08:04:02.2751 08:04:02 Same See note 1
Right Ascension (J2000) 13:16:20.142 13:16:20 Same See note 2
Declination (J2000) -08:04:01.31 -08:04:01 Same See note 2
Right Ascension (Precessed) 13:17:19.820 13:17:22 Diff See note 3
Declination (Precessed) -08:10:00.76 -08:10:15 Diff See note 3
Azimuth 103:09:10.3 103:09:12 Diff See note 4
Altitude +05:25:56.1 +05:25:43 Diff See note 4
Notes:

1 Differences in Sidereal Time are related to formatting. Internally, calculations use the full precision as seen in the MICA results.

2 The J2000 coordinates of the Sun are calculated by MICA to high precision. For the purpose of this test however I copied the MICA values into AstroCalc by using the Coordinates command dialog. By design the dialog limits data entry to whole numbers. As you can see the difference is very small but it will have an impact on further calculations albeit small.

3 As expected adjusting the J2000 coordinates for precession introduces some error in AstroCalc due to the complex nature of Precession. Looking at these numbers the difference is only about 20 arc-seconds which for most applications is tolerable.

4 The calculations of azimuth and altitude are affected by precession as described in the previous step but there is another source of error here: Horizontal Parallax. AstroCalc version 1 does not adjust for parallax. It will in a future version.

Final notes:

* MICA does not adjust for atmospheric refraction which affects altitude near the horizon. Neither does AstroCalc but it will in future versions.

* In the next version of AstroCalc you will be able to calculate the position of the Sun, Moon, and Planets using a new command button.

Conclusion

MICA is the gold standard as far as I am concerned. As mentioned in the notes above, adjusting for Horizontal Parallax will improve the results but there will always be some error mostly due to precession. Still I hope you find AstroCalc useful.
